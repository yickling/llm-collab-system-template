# Protocol Proposals

Append-only log where any developer can propose improvements to `PROTOCOL.md`.

## Rules

- Do **not** rewrite or delete historical proposals.
- Add newest proposals at the **top** of the Proposals section.
- Each proposal must include status and rationale.
- Mark adoption outcome after human decision.

## Proposal Template

Copy the block below when creating a new proposal:

```markdown
### [P-YYYY-MM-DD-NNN] Short title

- Date: YYYY-MM-DD
- Proposed by: (developer/agent name)
- Status: PROPOSED | ACCEPTED | REJECTED | SUPERSEDED
- Target section(s): (protocol section numbers/titles)
- Summary: (one paragraph describing the change)
- Proposed change:
  - (bullet 1)
  - (bullet 2)
- Rationale:
  - (reason 1)
  - (reason 2)
- Impact:
  - Benefits: (text)
  - Risks: (text)
- Human decision: PENDING | APPROVED | DECLINED
- Decision date: YYYY-MM-DD (if decided)
- Follow-up actions:
  - (action)
```
